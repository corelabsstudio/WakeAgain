데모 올리는 법

코딩·배포를 잘 몰라도 됩니다. 구매자와 검수자가 돌아가는 모습을 보면 됩니다.

한 줄 요약
공개 사이트 주소가 없으면 → 화면 녹화 1~2분 → YouTube/Loom/Drive 링크를 데모 칸에 붙이세요.
서버 올리기·스토어 등록은 필수가 아닙니다.

공통 (가장 쉬운 순서)

  1. 제품을 실행한다 (로컬·폰·에뮬 모두 OK)
  2. 화면을 녹화한다 (아래 OS별)
  3. 영상을 올린다 (유튜브 「링크 있는 사람만」, Loom, 구글 드라이브)
  4. 매물 등록의 데모 칸에 링크 + 한 줄 설명을 적는다

화면 녹화 어떻게 하나요?

  1. Windows — Win + G (Xbox 게임 바) → 캡처
  2. Mac — Cmd + Shift + 5 → 화면 기록
  3. iPhone — 설정 → 제어 센터 → 화면 기록 추가 후 스와이프해서 녹화
  4. Android — 빠른 설정 패널의 화면 녹화 (기종마다 이름 다름)

형태별로

  1. 웹사이트·SaaS — 주소가 있으면 주소. 없으면 브라우저 화면 녹화
  2. 모바일 앱 — 폰 화면 기록. TestFlight/내부 테스트는 보너스
  3. 데스크톱 — PC 화면 녹화. 설치 파일 첨부는 성사 후로 미뤄도 됨
  4. API — 호출 영상 또는 JSON 예시 + 문서 링크

검수에서 보는 것

  1. 데모가 실제로 열리는지 (링크 죽음 X)
  2. 상태(프로토타입 등)와 영상이 과하게 안 다른지
  3. 사기·명백한 도용 티가 없는지

코드 품질 감정·시세 감정은 하지 않습니다. “돌아가는 흔적”이 있으면 등록을 검토합니다.

내 프로젝트 올리기 상태 분류 기준
