## 2. Problem

### 섹션 제목

**세상 밖으로 나오지 못한 프로젝트들**

### 섹션 리드 (선언 문구 전문)

매일 수많은 프로젝트가 만들어집니다.

많은 제품들이 세상 밖으로 나오지 않습니다.

그들이 쓸모없어서가 아닙니다.

이는 인생이 변하거나, 시간이 없거나, 마케팅이 일어나지 않았기 때문입니다.

그래도 문제는 남습니다. 구체적으로는 이런 상황입니다.

### 문제 1 — 시간은 썼는데 수익·유저는 0

사이드 프로젝트 하나에 **주말 기준 8~12주**를 쓰는 경우가 많습니다.  
그런데 출시 후에도 **첫 유저 10명**을 못 넘기면, 서버·도메인 비용만 **매달 1만~5만 원**씩 나갑니다.  
“아깝다”와 “더 못 하겠다”가 동시에 남습니다.

### 문제 2 — 팔고 싶은데 장이 없다

커뮤니티·중고 거래에 올리면 문의는 오지만,  
**가격 기준·데모 검증·이전 절차**가 없어 **3~7일** 채팅 끝에 무산되기 쉽습니다.  
규모 있는 매각 플랫폼은 **수천만 원대 이상** 딜 위주라, 사이드 프로젝트는 문턱이 높습니다.

### 문제 3 — 사고 싶은데, 비교할 수 있는 매물이 없다

0부터 다시 만들면 MVP만 **4~8주**가 기본입니다.  
이미 손댄 코드를 사고 싶어도, 아이디어 PDF만 있는 매물과  
**실행 화면이 있는 매물**이 뒤섞여 있어 비교가 안 됩니다.  
입찰가·남은 시간이 안 보이면, “지금 사야 하나?”도 판단이 안 됩니다.

## 3. Services

### 섹션 제목 (선언 문구 중 해결 한 줄)

**우리는 그 프로젝트들에게 다시 한 번 기회를 줍니다.**

### 섹션 리드 (슬로건 재등장)

**프로젝트에 두 번째 기회를 주세요.**  
WakeAgain이 하는 일은 세 가지입니다.

### 서비스 1 — 잠든 프로젝트 등록 (판매)

데모·스토리·시작가·즉시구매가를 카드로 올립니다.  
**등록 비용 무료.** 검토 후 경매에 올리면, 폴더에만 있던 자산이 **공개 가격이 있는 매물**이 됩니다.

| 효과 수치 | 의미 |
|-----------|------|
| **등록 무료** | 올리기만으로 과금 없음 |
| **성사 시 약 10%** | 거래가 성사될 때만 수수료 |
| **24~48시간** | 접수 후 1차 확인 목표 시간(운영 기준) |

### 서비스 2 — 실시간 경매·입찰 (구매)

현재 입찰가와 **남은 시간(시:분:초)** 을 보고 입찰합니다.  
최소 입찰 단위 예: **+1만 원**. “얼마에 열리는지”가 숫자로 보입니다.

| 효과 수치 | 의미 |
|-----------|------|
| **입찰가 + 타이머 동시 표시** | 가격·긴급성을 한 화면에서 판단 |
| **최소 +₩10,000** | 호가 단위를 미리 명시(예시 정책) |
| **관심 등록 1회** | 맞는 매물 알림용 대기열 진입 |

### 서비스 3 — 검토 후 이전 (신뢰 절차)

데모·실행 흔적 있는 건을 우선합니다.  
핵심 소스는 **낙찰·합의 후** 이전. 연락 확인 → 합의 → 자산 인도 순서로 진행합니다.

| 효과 수치 | 의미 |
|-----------|------|
| **데모 우선 100%** | 아이디어만 있는 건은 전면 경매 후순위 |
| **이전 4단계** | 검토 → 입찰 → 합의 → 이전 |
| **웹·앱 1계정** | PC 사이트와 스토어 앱 데이터 공유 목표 |
