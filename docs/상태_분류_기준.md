# 제품 상태 분류 기준 (WakeAgain)

**목적:** 판매자가 상태를 고를 때 쓰는 공통 기준. 시작 입찰가·구매자 신뢰와 연결된다.  
**소스 of truth:** `wakeagain/pricing.py` · API `GET /api/v1/pricing` · 페이지 `/guide/status.html`

---

## 원칙

1. 상태는 마케팅 문구가 아니라 **완성도·리스크 라벨**이다.
2. **실행물(데모) 없는 아이디어 단독**은 등록 부적합.
3. 애매하면 **한 단계 낮게**(프로토타입 쪽) 잡는 것을 권장.
4. 과장 분류는 심사·분쟁 시 불리하다.

---

## 한눈에

| 상태 | 한 줄 | 권장 시작가 | 최저 |
|------|--------|-------------|------|
| 프로토타입 | 돌아가는 스케치 | 15만 | 5만 |
| 베타 | 써 보라고 줄 수 있는 제품 형태 | 40만 | 15만 |
| 출시됨·방치 | 세상에 나갔다가 멈춤 | 80만 | 30만 |
| 기타 | 앱/SaaS가 아닌 부품·자산 | 20만 | 5만 |

---

## 고르는 순서

1. 실행물 없으면 → 등록하지 말 것  
2. 스토어·공개 실서비스 이력 → **출시됨·방치** 검토  
3. 배포 URL은 있는데 정식 런칭 약함 → **베타**  
4. 부분 동작 스케치 → **프로토타입**  
5. 라이브러리·데이터·템플릿 등 → **기타**  

자세한 “이럴 때 / 아닐 때 / 데모” 목록은 API·가이드 페이지와 동일하다.
